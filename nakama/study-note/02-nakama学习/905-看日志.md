**结论**：你写的 `logger.Info(...)`、`logger.Error(...)` 都会输出到 **Nakama 的标准输出日志**，在你这个 `docker-compose` 部署里，就是 `nakama` 这个容器的日志。

### 1. 最直接：`docker-compose up` 直接看

在 `nakama/docker-compose.yml` 所在目录执行：

```bash
docker-compose up
```

- 终端里会实时看到 CockroachDB + Nakama 的所有日志。
- 插件加载成功时，会看到类似：

  ```text
  nakama  | 初始化 Nakama Go 模块...
  nakama  | demo_faction_get RPC registered
  nakama  | Nakama Go 模块加载完成
  nakama  | ...
  ```

你在 `login`、`demo` 里写的所有 `logger.Info(...)`、`logger.Error(...)` 都会混在这些输出里，因为 `docker-compose.yml` 里已经加了：

```text
--logger.level DEBUG
```

### 2. 已经在后台跑着时：单独看 Nakama 容器日志

如果你用 `docker-compose up -d` 后台启动了，可以单独看 `nakama` 服务的日志：

```bash
docker-compose logs -f nakama
```

- `-f` 会实时跟随输出（类似 `tail -f`）。
- 要退出按 `Ctrl + C` 即可。

### 3. 按关键字简单过滤（调试 demo/login 用）

想只看你模块相关的日志，可以给日志打统一前缀，例如：

```go
logger.Info("[demo] demo_faction_get called with payload: %s", payload)
logger.Info("[login] 用户登录成功，ID: %s", userID)
```

然后在日志里用管道过滤，例如（PowerShell）：

```powershell
docker-compose logs -f nakama | Select-String "\[demo\]"
```

这样就只会看到包含 `[demo]` 的行，方便你调试 `demo_faction_get` 之类的 RPC。
